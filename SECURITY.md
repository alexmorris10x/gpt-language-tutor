# Security Policy

## Supported versions

Security fixes are applied to the latest code on `main`. No older release line is currently supported.

## Report a vulnerability

Do not open a public issue for a suspected vulnerability. Use [GitHub private vulnerability reporting](https://github.com/alexmorris10x/gpt-language-tutor/security/advisories/new) and include reproduction steps, impact, affected files or versions, and any suggested mitigation.

Do not include a learner's private study records, voice transcripts, credentials, or personal data unless they are strictly necessary and safely redacted.

Reports are reviewed in good faith. Response and remediation timing depend on severity, reproducibility, and maintainer availability; this project does not promise a fixed response SLA.

Confirmed vulnerabilities will be documented through a GitHub security advisory or release notes when disclosure is appropriate.
